Klíčová hodnota není v samotném AI, ale ve sjednocení dat a procesů napříč holdingem. Z IT auditu a diskusí s vaším týmem víme, že datová kvalita v ERP QI je priorita č. 1 — bez ní nelze automatizovat reporting ani propojit entity. Proto začínáme právě tam.

1. Co je Discovery Workshop

Discovery Workshop je intenzivní 3týdenní fáze (15 pracovních dnů), během které společně:

  1. Identifikujeme a zvalidujeme 3–5 nejhodnotnějších AI use-casů napříč holdingem
  2. Ověříme technickou proveditelnost (přístup k datům, integrační možnosti, bezpečnost)
  3. Připravíme funkční demonstraci jednoho agenta na vzorku vašich reálných dat
  4. Připravíme roadmapu a rámec pro Fázi 1 (5 agentů)

Výstup Discovery je rozhodovací brána (Go/No-Go) — dostanete konkrétní data pro kvalifikované rozhodnutí, zda a jak pokračovat. Součástí je funkční PoC, ne jen PowerPoint.

6. Bezpečnostní architektura — třívrstvý model

Na základě vašich odpovědí z IT dotazníku a diskuse s Janem Ševčíkem navrhujeme třívrstvý bezpečnostní model. Citlivá výrobní data nikdy neopustí vaši infrastrukturu.

Vrstva 3 — Důvěrná (on-premise RD Rýmařov)

Data: Výrobní know-how, technologické postupy, konstrukční data, ceníky materiálů

Zpracování: Lokální LLM model (Qwen, Llama) — váš Mac Studio jako základ. Data nikdy neopustí vaši síť.

Vrstva 2 — Interní (EU cloud tenant s DPA)

Data: Obchodní data, reporting, CRM, dashboardy

Zpracování: Azure/AWS EU tenant s podepsanou Data Processing Agreement

Vrstva 1 — Veřejná (cloud LLM)

Data: Legislativa, veřejné registry, normy, monitoring certifikací

Zpracování: Cloud LLM (Claude, GPT-4) — nejlepší výkon pro netechnické úlohy

Klíčové zjištění z IT auditu: Infrastruktura je lokální, cloud pouze po detailní specifikaci bezpečnosti. Třívrstvý model toto respektuje. AI agenti mohou běžet hybridně — orchestrace v EU cloudu, citlivé operace nad výrobními daty lokálně. Přesnou klasifikaci dat odsouhlasíme na workshopu.

7. Klíčová technická zjištění z IT auditu

Na základě IT dotazníku a technické diskuse s vaším týmem jsme identifikovali tyto faktory ovlivňující architekturu řešení:

Příležitosti & připravenost

  • Testovací prostředí ERP QI k dispozici — můžeme bezpečně testovat bez rizika pro produkci
  • Git + CI/CD pipeline existuje — AI agenty nasadíme do stejného vývojového workflow
  • Monitoring k dispozici — napojíme na něj monitoring AI agentů
  • M365 stack (Teams, Exchange) — integrace přes Microsoft Graph API
  • Holdingová reportingová aplikace připravena — čeká na čistá data
  • Nový kalkulační systém ve vývoji — příležitost pro AI komponenty

Rizika & omezení k řešení v Discovery

  • ERP QI — integrační výzva č. 1. Legacy systém, API možnosti je třeba ověřit přímo s Melzerem
  • Nekonzistentní data v QI blokují reporting i konsolidaci
  • Tři oddělené identity systémy (M365, QI, nový systém) — žádné centrální SSO
  • IT kapacita podmíněna schválením — řešení musí být co nejvíce self-contained
  • Přechod CAD nástrojů (AutoCAD→ArchiCAD, Cadwork→SEMA) probíhá souběžně
  • Nejturbuletnější období inovací za 50 let historie — změny musí být koordinované
