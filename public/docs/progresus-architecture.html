Hrubý odhad technických komponent z 28 business agentů

VrstvaOdhadovaný početPoznámka
Orchestrátory~12-15Ne každý agent potřebuje orchestrátor. Jednoduché agenty jsou single-agent pattern.

Workflows~30-40Deterministické pipelines. Hodně z toho je ingestion, ETL, scheduled jobs — ne AI.

Klíčový messaging

Pro klienta: 28 agentů = 28 business výsledků. Nezajímá je architektura.

Pro nás interně: 28 business agentů = ~50-70 technických sub-agentů + ~30 workflows + ~20 shared skills. Tohle je scope, který musíme realisticky naplánovat.

Riziko: Pokud klientovi ukážeme technický rozpad, scope vypadá 3× větší a cena se bude zdát nízká — nebo naopak je vyděsíme komplexitou.

Co to znamená pro Progresus agenty

Tool layer (nejspodnější vrstva) se mění. Místo 1 MCP server per service můžeme mít 1 CLI pro celý Google Workspace. Orchestrátor zavolá gws gmail +send místo MCP tool call — výsledek je JSON, agent ho zpracuje stejně.

Výhoda pro Progresus: Snížení token cost o 10-30x na Google integracích. Jeden auth flow místo 5 separátních MCP serverů.

Pozor: gws je pre-v1.0, ne oficiálně supportovaný Google produkt. Pro produkční deploy zvážit hybrid — CLI pro interní tooling, MCP pro stabilní integrace.

Nenahrazuje: Custom MCP servery pro ERP/TPV/SAP. Ty jsou stále potřeba.

Strategy: Jak to stavět

Tier 1: Use existing (0 effort)

gws for Google Workspace, stripe for payments, gh for GitHub, Slack + Notion MCP (already connected)

Tier 2: Auto-generate (~2h each)

restish + OpenAPI spec. Point at HubSpot, Twilio, Jira, Linear. Get working CLI instantly. JSON output, agent-ready.

Tier 3: Lightweight custom (~1w)

Node/Python CLI wrapping REST API. Template: commander.js + axios. Real estate portals, internal tools.

Tier 4: Heavy custom (~2-4w)

ERP/SAP integration (RFC/BAPI), TPV system bridge, Legacy DB adapters. This is where Able's industry expertise matters.

Bottom Line: Dopad na Progresus projekt

Tier 1+2 CLIs pokryjí ~60% tool layer — Google Workspace, payments, CRM, communication, project management. Zero to minimal development effort.

Zbylých ~40% je ERP/TPV/legacy — custom build, ale to je stejné ať použijeme MCP nebo CLI. Rozdíl: CLI je levnější na tokeny a jednodušší na debug.

Doporučení: V Discovery fázi zmapovat OpenAPI specs u všech klientových systémů. Každý systém s OpenAPI = instant CLI za 2 hodiny místo 2 týdnů MCP serveru.

Riziko: gws je pre-v1.0. Pro produkční deploy doporučujeme hybrid — CLI pro dev/staging, fallback na direct API pro production.
